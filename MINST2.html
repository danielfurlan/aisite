import numpy as np
from sklearn.model_selection import train_test_split
from tensorflow import keras
import json
import ast

img_rows, img_cols = 28, 28
num_classes = 10

def prep_data(raw):
    y = raw[:, 0]
    out_y = keras.utils.to_categorical(y, num_classes)
    
    x = raw[:,1:]
    num_images = raw.shape[0]
    out_x = x.reshape(num_images, img_rows, img_cols, 1)
    out_x = out_x / 255
    return out_x, out_y

fashion_file = "../input/fashionmnist/fashion-mnist_train.csv"
fashion_data = np.loadtxt(fashion_file, skiprows=1, delimiter=',')

from tensorflow import keras
from tensorflow.keras.models import Sequential
from tensorflow.keras.layers import Dense, Flatten, Conv2D

fashion_model = Sequential()

fashion_model.add(Conv2D(12,kernel_size=(3,3),activation="relu", input_shape=(img_rows,img_cols,1)))

The first layer is a Conv2D layer. It has 12 filters, a kernel_size of 3 and the relu activation function. The first layer always requires that you specify the input_shape..

  1. I'm gonna add 2 more convolutional (Conv2D layers) with 20 filters each, 'relu' activation, and a kernel size of 3. Follow that with a Flatten layer, and then a Dense layer with 100 neurons. The Flatten layer is to reshape our input that has become a tensor with higher dimension than the input of the neurons. This is why we use such a layer.
  2. Finally, I add the prediction layer, a Dense layer. As we have 10 possible outputs, this is the number of neurons.

What is interessant to observe :

  1. we have better accuracy for small kernel_sizes. Tipically, the best results come from tuples (2,2) < (3,3). We could imagine, in a first analyses, that this is normal, as smaller kernel_sizes will have to read the intere image with smaller steps (will have more steps to cross all the rows and columns)
  2. for bigger values it seems that we have to increase the number of filters (it is in accordance with the previous point .1) if we want to achieve the same accuracy
  3. the number of layers in this case (in the way the model was built) doesn't have a strong positive correlation with accuracy. It is taked for granted that higher number of layers improve the model performance. And this is what we certainly would observe more clearly here if the number of layers were superior than only 4. But we can have good accuracy values here with fewer layers.
  4. Basically, if we have small kernel sizes, it will be necessary less layers and filters for a given accuracy. Otherwhise, we can expect to increase the number of filters and layers. Anyway, I don't think that this relationship must be seen as linear and always true, as the complexity of tasks and images data can really differ and change the output. This is by the way what we can observe above
